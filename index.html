Quick Reference — Day One Cheat Sheet
1
Paste the message
Drop the patient's message into the left box. If it's a reply to your earlier questions, click + Prior context first and paste the earlier thread there.
2
Run Triage
The AI reads the message + your full Knowledge Base and returns a classification, severity, routing flags, and a draft response — usually in 5-10 seconds.
3
Review and act
Check the severity badge and routing cards first — these tell you if anything needs to be escalated before you send a response. Then use or edit the draft.
4
Teach the system
After you send your response, paste what you actually sent into What Did You Actually Send? — even if it was identical. This is how the AI learns.
Reading the Output Cards
The colored badge — red, yellow, or green — tells you how serious the reported side effect is based on your clinical KB rules. Red (Severe) means escalate to a provider immediately. Yellow (Moderate) means gather more information before deciding. Green (Mild) means an education response is appropriate. If there are no side effects, no badge appears. Always check this before reading the draft.
A small gray bar at the top of every result shows exactly what the AI classified: the clinical category, side effect severity (if detected), and response timeframe. Check this first to confirm the AI understood the message correctly. If anything looks wrong, expand the Correct, vote, or teach the AI section at the bottom to fix it.
When the AI detects a non-clinical item (billing, shipping, account), a routing card appears above the draft response. It shows what needs to go to the support team and contains an Internal Note for Staff — paste this into the Bask chat thread so the assigned staff member knows exactly what to do. The draft response handles the patient-facing communication and automatically acknowledges that the non-clinical item is being routed.
The timeframe dropdown shows the AI's suggested urgency — Routine, Same Day, Urgent. It maps to your KB urgency rules. You can change it using the dropdown and save the correction. Urgent means escalate now. Same Day means provider review needed today. Routine means a standard response queue is fine.
Using Prior Context
Any time the patient message is a reply to something you already sent. If a patient is responding to your follow-up questions, their reply alone may be only a few sentences with almost no context. Without the prior thread, the AI is triaging a fragment. Click + Prior context, paste the earlier conversation, then paste their latest reply in the main box. The AI will read the full thread and respond to the latest message accordingly.
Prior Context: Everything that happened before the current message — the patient's first message, your initial response, any earlier back-and-forth. The AI reads this for background only and does not respond to it directly.

Main Message box: The patient's most recent message — the one you are triaging right now. The AI responds to this. The label changes to "Latest Reply" when prior context is active as a reminder.
No. For a patient's first message — one that is not a reply to anything — leave Prior Context empty. It adds tokens to the AI call and slightly increases processing time, so only use it when the current message lacks enough context to triage on its own.
Teaching the AI — the Learning Loop
This field is how the AI learns from real clinical decisions. After you send your response to the patient, paste what you actually sent here — but only when you changed the draft. If you sent the draft verbatim, you can skip this field. When there is a difference between the draft and what you sent, the AI analyzes what changed and saves a learning note. Over time these corrections help the AI produce drafts that are closer to how you actually write.
Go to Clinical Knowledge Base → Corrections tab and click Load Response Corrections. Find the incorrect entry and click Delete. This clears the mistaken correction from the database so it does not influence future responses. The underlying triage record is preserved — only the correction is removed.
The Clinical Category and Non-Clinical dropdowns in the Classification card are editable. If the AI mis-tagged a message — for example, categorizing a vomiting question as "General" when it clearly belongs to "GI side effects," or tagging a dosing question as "Injection/administration" when it should be "Dosing question" — fix it and click Save. Each correction is stored in the database and builds a dataset of right vs. wrong classifications that can be reviewed to improve KB rules over time.
Voting records specific feedback about AI performance on each triage. Select as many reasons as apply before submitting — this is richer data than a single rating. The votes are stored in the database and can be reviewed to find patterns: if the AI is consistently missing dose checks, or getting tone wrong, or mis-routing — that signal should be addressed by updating the corresponding KB rule or protocol.
Managing the Knowledge Base
The Knowledge Base is the AI's memory. Every time you run a triage, the AI reads the entire KB — protocols, side effect guidance, routing rules, urgency rules — before processing the patient message. This means any edit you make takes effect immediately on the next run. The KB is shared across all nurses on your team. There is no retraining involved — it is just a document the AI reads.
Add an entry when you find yourself writing the same type of response repeatedly and the AI is not producing it correctly. Good candidates: a medication tip you give to every plateau patient, a specific protocol for a patient population you see often, a routing rule for a new department, a URL you reference constantly. If you had to edit the draft significantly, that is a signal — add a note to the KB explaining what the response should look like.
Side Effects — education and management for any side effect patients report.
Templates — response frameworks for common non-emergency messages like plateau questions or food noise.
Protocols — if/then decision trees: "if symptom X and condition Y, do Z."
Rules and Notes — urgency thresholds, severity classification, dose verification rules. Be careful editing these — they govern how the AI routes and classifies everything.
Routing — rules about when to escalate clinically or route non-clinically.
URLs — links you reference in patient responses.
Entries under Rules and Notes and Routing govern the AI's core behavior — urgency classification, severity levels, routing logic. Editing these carelessly can cause the AI to mis-classify or miss escalations. Before editing a rules entry, read it fully first. If you are unsure, add a new entry rather than modifying an existing one, and test it on a few sample messages. You can always delete a new entry if it causes problems.
Common Mistakes to Avoid
Triaging a patient reply without Prior Context — the AI gets a fragment with no history and produces a generic response.

Skipping "What Did You Actually Send?" — the learning loop does not run. The system cannot improve without this data.

Entering the wrong response in the correction field — this teaches the AI the wrong thing. If you make a mistake, delete it in the Corrections tab immediately.

Ignoring the red severity badge — a severe badge means escalate now, not send an education response. Always check routing cards before using the draft.
