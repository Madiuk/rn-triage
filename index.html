Quick Reference — How This Works
1
Paste the patient message
Drop the patient's current message into the left box. If it's a reply to something you already sent, click Add Prior Context and paste the earlier thread there first.
2
Click Run Triage
The AI reads the message against your Knowledge Base and returns a classification, routing decision, and a draft response. Usually takes 5–10 seconds.
3
Check the results top to bottom
Classification strip — what the AI decided. Routing card — only appears when something needs to go to another team; copy the internal note into Bask. Generated Response — copy and send to patient.
4
Teach the AI if you edited
If you changed the draft before sending, paste what you actually sent into I Edited Before Sending and click Submit & Learn. Only do this when you made changes — not every time.
Reading the Results
The gray bar at the very top of every result shows what the AI decided at a glance: the clinical category (e.g. GI side effects), the response timeframe (Routine / Same Day / URGENT), and a colored side effect severity badge if a side effect was detected. Check this first. If anything looks wrong, scroll down to the Message Category or Response Timeframe cards to correct it.
The colored badge appears only when a side effect is detected. Side Effect: Severe means escalate to a provider immediately — do not wait for the patient to reply. Side Effect: Moderate means gather more information in your initial response and reassess. Side Effect: Mild means an education response is appropriate. No badge means no side effect was detected.
The routing card only appears when the AI detects a non-clinical item — billing, shipping, account issues, prescription transfers, etc. It shows what needs to go to the support team and contains an Internal Note for Staff: paste this directly into the Bask chat thread so the assigned staff member knows exactly what to investigate. The draft response automatically acknowledges to the patient that their non-clinical item is being handled.
When a message contains both a clinical question and a non-clinical item (e.g. asking about nausea AND a shipping delay), the routing card shows a Dual Task header. The draft response handles the clinical portion. The internal note handles the non-clinical routing. You copy both separately — the response to the patient, the note into Bask for the support team.
The timeframe tells you how urgently the patient needs a reply. Routine means respond in normal queue order. Same Day means the patient needs a reply today. URGENT means escalate now — do not put this in the queue. If the AI got it wrong, scroll down to the Response Timeframe card, adjust the dropdown, and click Save.
Using Prior Context
Any time the patient message is a reply to something you already sent. A patient answering your follow-up questions may write only 2–3 sentences with almost no context on their own. Without the earlier thread, the AI is triaging a fragment and the response will be generic. Click Add Prior Context, paste everything that came before, then paste only their latest reply in the main message box.
Prior Context box: The entire earlier conversation — the patient's first message, your response, any back-and-forth. The AI reads this for background only and does not respond to it.

Main message box (labeled "Latest Reply" when prior context is active): The patient's most recent message only. This is what the AI triages and drafts a response to.
No. For a patient's first message — one that isn't a reply to anything — leave Prior Context empty. Only use it when the current message would make no sense without knowing what came before it.
Teaching the AI
After sending your response to the patient, if you changed the draft, paste what you actually sent in the I Edited Before Sending field (inside the Generated Response card) and click Submit & Learn. The AI compares the two versions, identifies what changed, and saves a learning note. Over time, these corrections shift how the AI drafts responses to similar messages. If you sent the draft verbatim, skip this entirely.
Scroll to the Message Category card below the response. Click the correct category in the Clinical or Non-Clinical list (click again to deselect the wrong one), then click Save Categories. Each correction is stored and builds a dataset that can be reviewed to improve the AI's classification accuracy over time.
It appears in the Message Category card when a severity badge was triggered. If the AI incorrectly flagged something as a side effect — for example, a food noise or plateau question triggering a "Mild" badge — check this box and click Save Categories. This records a false positive in the database so the pattern can be addressed.
The voting buttons appear directly under the draft response. They record qualitative feedback about AI performance. Select as many reasons as apply and click Submit. Votes are stored in the database and can be reviewed to spot patterns — if the AI is consistently getting tone wrong, or missing routing decisions, that signal should be addressed by updating the relevant Knowledge Base entry.
Go to Clinical Knowledge Base → Corrections tab and click Load Response Corrections. Find the incorrect entry and click Delete. This removes the mistaken correction from the database. The underlying triage record is preserved — only the correction note is cleared.
Knowledge Base
The Knowledge Base is everything the AI reads before responding to a patient message. It contains your clinical protocols, side effect guidance, routing rules, urgency thresholds, and response templates. Any change you make takes effect immediately on the next triage. It is shared across all staff on your team.
Add an entry when the AI keeps missing something you know well — a specific response pattern, a protocol you use often, a routing rule for a new scenario, or a URL you reference regularly. If you edited the draft significantly for the same type of message twice, that's a signal to document it in the KB.
Side Effects — education and management guidance for patient-reported side effects.
Templates — response frameworks for common non-emergency messages (plateau questions, food noise).
Protocols — if/then decision logic: if symptom X and condition Y, do Z.
Rules & Notes — urgency thresholds, severity rules, dose verification. Edit carefully — these govern everything.
Routing — when and how to escalate or route messages.
URLs — links you reference in patient responses.
Common Mistakes
Triaging a reply without Prior Context — the AI gets a fragment and produces a generic, often unhelpful response.

Ignoring the red severity badge — Severe means escalate to a provider now, not draft an education response.

Ignoring the routing card — if a non-clinical item is flagged, the internal note needs to go into Bask before the thread is reassigned.

Entering the wrong thing in "I Edited Before Sending" — this teaches the AI incorrect behavior. Delete it in the Corrections tab immediately.

Editing KB rules entries without reading them fully first — the Rules & Notes and Routing entries govern how the AI classifies every message. Small changes have broad effects.
